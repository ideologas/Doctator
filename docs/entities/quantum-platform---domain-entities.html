Quantum Platform - Domain Entities

User

Represents a registered, authenticated individual or business entity interacting with the platform. A user has a profile, credentials, and associated context for personalized experiences.

Links

No. Name Entity Cardinality Description
LI-01 Owns Access Key 1..1 Each user has one Access Key that stores their context for personalization.
LI-02 Undertakes Journey 0..* A user can initiate and progress through multiple journeys.

Fields

No. Name Type Mandatory Description
FD-01 Id guid Y Unique identifier for the user.
FD-02 Email text Y The user's unique email address, used for login.
FD-03 PasswordHash text Y The hashed password for authentication.
FD-04 FirstName text N The user's first name.
FD-05 LastName text N The user's last name.
FD-06 CompanyName text N The name of the company the user represents, if applicable.
FD-07 Role text Y The role of the user (e.g., 'individual', 'business'). Used for personalization.
FD-08 CountryCode text Y The user's country code (e.g., 'lt', 'us').
FD-09 PhoneNumber text N The user's phone number.
FD-10 RefreshToken text N The token used to refresh the authentication session.
FD-11 RefreshTokenExpiryTime timestamp N The expiration date and time for the refresh token.

Anonymous User

Represents a temporary, non-authenticated visitor to the platform. An anonymous session is created to track user context and journey progress before they register or log in.

Links

No. Name Entity Cardinality Description
LI-01 Owns Access Key 1..1 Each anonymous user has one Access Key that stores their temporary context. This key is merged upon login.
LI-02 Undertakes Journey 0..* An anonymous user can start and progress through journeys, which can be linked to their account upon registration.

Fields

No. Name Type Mandatory Description
FD-01 UserId guid Y Unique temporary identifier for the anonymous session.
FD-02 RefreshToken text Y The token used to refresh the anonymous authentication session.
FD-03 RefreshTokenExpiryDate timestamp Y The expiration date and time for the refresh token.

Access Key

A dynamic set of key-value pairs and flags representing a user's current context. It is used by the Content Service to filter and personalize the content delivered to the user. It is the central piece of the personalization engine.

Links

No. Name Entity Cardinality Description
LI-01 Owned by User 1..1 An Access Key is associated with a single registered User.
LI-02 Owned by Anonymous User 1..1 An Access Key is associated with a single Anonymous User session.
LI-03 Used by Access Group *..* The key-value pairs and flags are evaluated against Access Group rules to determine content visibility.

Fields

No. Name Type Mandatory Description
FD-01 KeyValuePairs dictionary<text, text> Y A collection of key-value pairs describing the user's context (e.g., 'country:lt', 'role:individual').
FD-02 WordingFlags list<text> Y A list of flags indicating user state or choices (e.g., 'JourneyStarted', 'question_key-answer_key').

Tenant

Represents a top-level instance of the application, with its own branding, domain, configuration, and content. It provides multi-tenancy for the platform.

Links

No. Name Entity Cardinality Description
LI-01 Has Navigation Item 0..* A Tenant has a main navigation menu composed of multiple navigation items.
LI-02 Defines Page *..* A tenant has a set of pages that constitute its website or application.

Fields

No. Name Type Mandatory Description
FD-01 Key text Y A unique string key that identifies the tenant (e.g., 'metasite', 'exacaster').
FD-02 AppName text Y The display name of the application for this tenant.
FD-03 Logo media N The logo image for the tenant.
FD-04 ThemeColor text N The primary theme color for the tenant's UI.
FD-05 Description text N A brief description of the tenant.
FD-06 AppNameInHeader boolean Y Whether to display the AppName in the header.

Page

A fundamental unit of content representing a screen or view in the application. It acts as a container for other content components like panels and banners, and can be part of a user journey.

Links

No. Name Entity Cardinality Description
LI-01 Contains Panel 0..* A page is composed of one or more panels that structure its content.
LI-02 Filtered by Access Group 0..* Access groups control the visibility of the entire page for different users.
LI-03 Can be part of Journey 0..1 A page can be a step within a defined user journey.

Fields

No. Name Type Mandatory Description
FD-01 Key text Y A unique, human-readable key for the page (e.g., 'landing', 'login').
FD-02 Title text Y The display title of the page, often shown in the browser tab.
FD-03 URL text Y The URL path for accessing the page (e.g., '/', '/login').
FD-04 BackPageKey text N The key of the page to navigate to when a 'back' action is performed.

Journey

Represents a stateful, multi-step process or workflow that a user undertakes. It tracks the user's progress through a sequence of pages (steps).

Links

No. Name Entity Cardinality Description
LI-01 Undertaken by User 1..1 Each journey instance is associated with a specific user.
LI-02 Consists of Journey Step 1..* A journey is composed of a sequence of steps.

Fields

No. Name Type Mandatory Description
FD-01 Type text Y The type of journey (e.g., 'onboarding', 'insurance_quote').
FD-02 UserId guid Y The unique identifier of the user undertaking the journey.
FD-03 Status text Y The current status of the journey (e.g., 'Started', 'Submitted', 'Completed').
FD-04 StartDate timestamp Y The date and time when the journey was started.
FD-05 SubmissionDate timestamp N The date and time when the journey was submitted.
FD-06 ExpirationDate timestamp N The date and time when the journey will expire.
FD-07 WordingFlags text N A semicolon-separated list of flags accumulated during the journey.

Journey Step

A single step within a Journey, representing a user's transition from one page to another. It can store data captured at that step.

Links

No. Name Entity Cardinality Description
LI-01 Part of Journey 1..1 Each step belongs to a single journey instance.
LI-02 Can have Question Form 0..1 A step can represent the submission of a question and its answer.

Fields

No. Name Type Mandatory Description
FD-01 SequenceNumber integer Y The order of this step within its journey branch.
FD-02 CurrentPageKey text Y The key of the page where the step was initiated.
FD-03 NextPageKey text Y The key of the page the user navigated to, completing the step.
FD-04 SubmitDate timestamp Y The date and time the step was submitted.
FD-05 JourneyGenericDataList json N A list of generic JSON data captured during this step.

Question Form

Represents a user's answer to a specific question within a journey. This is a specialized type of Journey Step data.

Links

No. Name Entity Cardinality Description
LI-01 Belongs to Journey Step 1..1 Each Question Form is data for a single journey step.

Fields

No. Name Type Mandatory Description
FD-01 QuestionKey text Y The unique key identifying the question being answered.
FD-02 AnswerKey text Y The key representing the selected answer.
FD-03 AnswerValue text N Optional additional value associated with the answer (e.g., free text input).

Persona

A defined user profile or segment based on a set of attributes. Personas are used in Access Groups to control content visibility for different types of users.

Links

No. Name Entity Cardinality Description
LI-01 Used in Access Group *..* A Persona can be used in multiple Access Groups to define show/hide rules.

Fields

No. Name Type Mandatory Description
FD-01 Key text Y Unique key for the persona (e.g., 'AuthenticatedBusinessUser').
FD-02 WordingFlags text N A semicolon-separated list of flags that must be present in the user's Access Key.
FD-03 IsAuthenticated enumeration N Specifies if the user must be authenticated ('Yes', 'No', or 'Any').
FD-04 Tenant enumeration N The tenant the user must belong to.
FD-05 Role enumeration N The role the user must have.
FD-06 Country enumeration N The country the user must be from.

Access Group

A container for a set of rules that defines content visibility. It works by including a list of 'ShowTo' personas and excluding a list of 'HideFrom' personas. Content is visible if the user matches a 'ShowTo' persona and does not match any 'HideFrom' persona.

Links

No. Name Entity Cardinality Description
LI-01 Applies to Page *..* An Access Group can control the visibility of multiple pages.
LI-02 Includes Persona 0..* An Access Group is defined by a set of Personas.

Fields

No. Name Type Mandatory Description
FD-01 Key text Y Unique key for the access group.
FD-02 ShowTo relation N A list of Personas. A user must match at least one of these to see the content.
FD-03 HideFrom relation N A list of Personas. If a user matches any of these, they will not see the content, regardless of the ShowTo list.

Panel

A structural component within a Page used to group and organize content. It can be divided into multiple columns with specified distributions.

Links

No. Name Entity Cardinality Description
LI-01 Contained in Page *..1 A panel is a component of a single page.

Fields

No. Name Type Mandatory Description
FD-01 Key text N A unique key for identifying the panel.
FD-02 Title text N An optional title for the panel.
FD-03 ColumnDistribution enumeration N The layout of columns within the panel (e.g., 'single', 'two-1:1', 'two-2:1').
FD-04 Columns component N The collection of column components that hold the actual content items.
FD-05 BackgroundColor text N The background color of the panel for the light theme.
FD-06 BackgroundColorDarkTheme text N The background color of the panel for the dark theme.

A reusable block of content, typically containing rich text (Markdown) and a title. It's a fundamental building block for displaying formatted text and images within pages.

Links

No. Name Entity Cardinality Description
LI-01 Can be part of Page *..* A content block can be placed on multiple pages via panels and columns.

Fields

No. Name Type Mandatory Description
FD-01 Key text Y A unique key identifying the content block.
FD-02 Title text N The title of the content block.
FD-03 Content richtext N The main body of content, supporting Markdown and rich text formatting.
FD-04 TitleAlignment enumeration N The horizontal alignment of the title (left, center, right).
FD-05 ContentAlignment enumeration N The horizontal alignment of the content (left, center, right).
